加速 Git Clone
发表于2020-07-12|更新于2020-08-08|Github
|字数总计:299|阅读时长:1分钟|阅读量:

原因：Git clone 特别慢是因为 github.global.ssl.fastly.Net 域名被限制了。只要找到这个域名对应的 ip 地址，然后在 hosts 文件中加上 ip–> 域名的映射，刷新 DNS 缓存就可以了

1. 查询域名 global-ssl.fastly.Net 和 github.com 公网地址

https://www.ipaddress.com/ 可以用这个查。

或者 Dns检测|Dns查询 - 站长工具 查找检测列表里的TTL值最小的IP

分别查找github.global.ssl.fastly.net和github.com这两个网址的 ip 地址

2. 将 ip 地址添加到 hosts 文件

Linux：sudo gedit /etc/hosts
Windows：C:\Windows\System32\drivers\etc\hosts

保存退出

3. 刷新 DNS 缓存

修改完 hosts 还不会立即生效，你需要刷新 DNS 缓存，告诉电脑我的 hosts 文件已经修改了。

Linux：sudo /etc/init.d/networking restart

windows 下刷新 DNS 的方法：

打开 CMD

输入ipconfig /flushdns

重新 clone 就会发现速度飞起

如果不行就把github.global.ssl.fastly.net的配置删掉，只留github.com
